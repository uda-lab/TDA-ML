# 公開可否判定基準（F-1）

## 目的

- 本書を公開前ゲート判定の正本とし、公開可否を機械的に判定可能な状態に固定する。
- 既存の A-1 分類（`public/private/external/needs-owner-decision`）と矛盾する場合、公開可否判定は本書の F-1 基準を優先する。

## 判定クラス

### 公開可

以下をすべて満たす場合に限り `公開可` とする。

- CI最小要件: `uv run ruff check .` が成功する（exit code 0）。
- 秘密情報: `.env`、鍵、トークン、認証情報ファイルが Git 管理されていない。
- 巨大ファイル: 10 MiB 超ファイルが Git 管理されていない。
- 外部依存: 外部リポジトリ参照方針と再現手順が文書化され、再配布禁止物を含まない。

### 条件付き公開

必須ゲートは満たすが、公開範囲に owner の最終判断が必要な項目が残る場合は `条件付き公開` とする。

- 1 MiB 超 10 MiB 以下ファイルを公開対象へ含める必要がある。
- 生成物（`outputs/`）を一部公開するが、再生成可能性・容量理由・公開範囲の記録が未確定。
- 外部データや外部実装の再配布可否が契約/ライセンス確認待ち。

### 公開不可

以下のいずれかに該当した場合は `公開不可` とする。

- CI最小要件が未実行、または失敗している。
- 秘密情報混入が確認された、または疑義を解消できない。
- 10 MiB 超ファイルが Git 管理されている。
- 外部依存・外部データの再配布可否が不明なまま公開しようとしている。

## 判定ルール（カテゴリ別）

### 秘密情報

- Git 管理禁止対象: `.env`, `*.pem`, `*.key`, `*.p12`, `credentials*.json`, トークン文字列を含む設定・ログ。
- 履歴を含むスキャンで疑義が残る場合は即時 `公開不可` とし、`要owner判断` に送る。
- マスク済み情報のみが残る場合に限り判定を継続できる。

### 巨大ファイル

- 10 MiB 超: Git 管理禁止（例外なし、`公開不可`）。
- 1 MiB 超 10 MiB 以下: 原則除外。再現性に必須な場合のみ `要owner判断` を経て `条件付き公開`。
- 生成物の同梱は原則禁止。再生成不能な根拠がある場合のみ owner 承認で例外可。

### 外部依存

- `ellphi_repo/`, `pytorch-topological/` など外部依存は、責務境界と参照方法（取得元・ライセンス）を明記する。
- 外部依存の commit 固定方式が未確定な場合、再現責任が不明確なため `条件付き公開` 以下とする。
- 再配布禁止の外部資産を含む場合は `公開不可`。

## 要owner判断

- 生成物公開範囲の境界（`outputs/` をどこまで公開対象に含めるか）。
- 1 MiB 超 10 MiB 以下の容量例外を認めるか。
- `docs/JSAIM2026UT/JSAIM2026UT.pdf` や `docs/paper/` 配下資料の再配布可否。
- 外部データ・外部実装の再配布可否（契約・ライセンス適合）。
- 外部依存の固定方針（gitlink 固定、commit 記録、参照のみ）をどれに統一するか。

## 既存分類（A-1）との関係

- A-1 の `public/private/external/needs-owner-decision` は棚卸し分類として維持する。
- 公開可否判定は本書 F-1 を正とし、分類と判定が衝突した場合は F-1 判定を採用する。
